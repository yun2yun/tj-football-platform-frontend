## 三、核心功能清单（P0 必做，无遗漏）

### 3.1 登录模块

表格

|功能点|详细规则|
|---|---|
|账号输入|必填，支持数字 / 字符串，无长度限制（模拟场景）|
|密码输入|必填，明文展示（模拟场景，无需加密）|
|角色选择|必填，下拉框仅展示「学生 / 教师」，默认空|
|登录校验|仅校验账号 / 密码 / 角色是否为空，不校验真实性（模拟登录）|
|登录跳转|学生跳转到「学生成果列表页」，教师跳转到「教师待审核列表页」|
|登录态存储|登录成功后，将用户信息（账号、姓名、角色）存入 localStorage|

### 3.2 学生端模块

表格

|功能点|详细规则|

## 五、页面交互与校验规则（精准到操作）

### 5.1 通用交互规则

1. 所有按钮点击后有「加载中」状态（模拟，无实际请求）；
2. 操作成功 / 失败均弹出 Element Plus 的 Message 提示框，提示语简洁（如「提交成功」「审核成功」）；
3. 页面跳转使用 vue-router 的 push 方法，无刷新跳转；
4. 侧边栏选中项高亮显示，与当前路由匹配。

### 5.2 表单校验规则

表格

|字段|校验规则|错误提示语|
|---|---|---|
|账号|非空|请输入账号|
|密码|非空|请输入密码|
|角色|非空（下拉框必选）|请选择登录角色|
|成果名称|非空 + 长度 2-50 字|请输入 2-50 字的成果名称|
|成果类型|非空（下拉框必选）|请选择成果类型|
|成果描述|非空 + 长度 10-500 字|请输入 10-500 字的成果描述|
|审核意见（驳回）|非空 + 长度≥5 字|驳回时请填写至少 5 字的审核意见|

### 5.3 权限控制规则（前端模拟）

1. 未登录用户访问学生 / 教师页面，自动跳转到登录页；
2. 学生角色访问教师页面，弹出「无权限访问」提示，跳转到学生首页；
3. 教师角色访问学生页面，弹出「无权限访问」提示，跳转到教师首页；
4. 页面刷新后，从 localStorage 读取用户信息，保持登录状态；
5. 退出登录后，清空 localStorage 并跳转登录页。
