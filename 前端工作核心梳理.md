## 一、项目核心定位（你要做的前端是为这个平台服务）

这个平台是**同济大学国际足球学院专属的创新创业成果记录网页平台**，核心用于学生提交足球相关创新成果、教师审核、管理员管理，最终对接学院的创新创业学分认定工作，是纯 PC 端的管理类网页，核心要实现**成果申报、分级审核、记录存档、数据查询**四大核心功能，且平台仅做成果记录，不涉及学分评分。

## 二、目前已完成的基础工作（团队 + 你个人）

### 团队层面

1. 已明确项目整体技术架构：前后端分离架构 + Web 三层架构，分学生 / 教师 / 管理员三个角色设计权限；
2. 已完成需求调研、业务梳理：明确了成果分类、提交要求、审核流程，也敲定了平台的功能边界；
3. 有硬件资源支撑：指导老师提供了服务器、数据库，后端开发的基础环境已具备。

### 你个人层面

1. 已成功搭建 Vue3+Vite 的前端基础项目框架，且安装了开发所需的 Element Plus（UI 组件）、axios（接口请求）、vue-router@4（路由）核心依赖；
2. 项目文件夹已和 Gitee 仓库关联，具备代码同步、团队协作的基础。

## 三、队友对你的核心要求（当前优先级最高）

队友负责后端开发，要求你**设计好前端所需的所有接口并提供给他们**，这是前后端协作的核心衔接点，且你可以选择寒假开展具体的前端页面开发，但**接口设计需要优先交付**，后端会根据你设计的接口进行开发，避免前后端对接脱节。

## 四、你接下来的核心工作（分优先级，无代码纯落地思路）

### 优先级 1：核心 —— 设计前端所需的全量接口（需交付给后端）

结合平台的核心功能和三个角色的权限，梳理出前端所有操作对应的接口，只需明确接口的核心信息，不用写代码，整理成清晰的文档 / 表格即可，核心要包含：

1. 接口对应的**功能场景**（如学生端提交成果、教师端审核成果、管理员端查询成果列表）；
2. 接口的**请求方式**（如查询数据用 GET、提交 / 修改数据用 POST/PUT、删除用 DELETE）；
3. 接口的**核心参数**（如学生提交成果需要传成果名称、类型、附件、提交人信息等；教师审核需要传审核结果、审核意见、审核人 ID 等）；
4. 接口的**返回数据需求**（如前端查询成果列表，需要后端返回成果的所有信息、审核状态、提交时间等）；
5. 按**角色 / 功能模块分类**设计接口（如学生模块、教师审核模块、管理员管理模块），方便后端对接和后续维护。

### 优先级 2：基础 —— 规划前端项目的整体结构（为后续开发做准备）

结合平台的三个角色和核心功能，提前规划前端项目的文件 / 页面结构，让后续开发更规范，核心梳理：

1. 页面分层：按角色分学生端页面、教师端页面、管理员端页面，核心页面包含登录页、成果申报页、成果列表页、审核页、用户管理页等；
2. 公共部分：规划好公共的导航栏、侧边栏、页脚、权限校验组件（如不同角色看到不同的页面 / 功能）；
3. 功能模块：将申报、审核、查询、管理等功能拆分成独立的模块，避免后续代码混乱。

### 优先级 3：可选 —— 寒假开展前端具体开发（按自己的时间安排）

接口交付给后端后，你可以利用寒假进行前端的实际开发，核心开发节奏：

1. 先做**通用基础页面**：如登录页、公共的布局页面（导航 / 侧边栏），实现不同角色的登录后跳转；
2. 再按**角色模块开发核心页面**：先开发学生端的成果申报、成果查询页（核心功能），再开发教师端的审核页，最后开发管理员端的管理页；
3. 开发过程中**同步对接后端**：后端完成部分接口后，可提前进行接口联调，确保功能正常。

### 优先级 4：辅助 —— 同步代码 + 沟通确认（保障团队协作）

1. 接口设计过程中，**及时和后端队友沟通确认**：如接口的参数、返回数据是否合理，是否符合后端的开发规范，避免后续反复修改；
2. 所有的接口设计文档、项目规划内容，都**同步到 Gitee 仓库**，方便团队成员查看；
3. 后续开发的代码，按功能模块及时提交、推送，保持代码同步。

## 五、核心注意点（贴合项目特色，避免踩坑）

1. 接口设计要**贴合足球专业特色**：结合团队梳理的足球创新成果分类标准，设计成果申报、查询的接口参数，确保适配学院的专业需求；
2. 接口设计要**考虑角色权限**：如学生只能操作自己的成果，教师只能审核自己负责的成果，接口中要包含权限校验的相关参数（如用户 ID、角色标识）；
3. 接口设计要**预留扩展空间**：项目要求平台具备高扩展性，后续可能会对接学分认定功能，设计接口时避免写死参数 / 返回数据，预留后续扩展的字段；
4. 所有设计内容**形成文档**：接口设计文档、前端项目规划文档都要整理好，不仅方便后端对接，也为后续自己开发、项目维护提供依据。
