## 项目简介

**同济足球创新成果记录平台（前端简易版）**，用于为同济大学国际足球学院搭建一个 PC 端网页平台，支持学生提交足球相关创新成果、教师进行单级审核。当前仓库为**纯前端静态原型**，通过本地 `localStorage` 模拟数据，方便在无后端环境下演示完整业务闭环。

> 说明：正式版的前后端接口已在 `前端接口设计文档.md` 中设计，本项目的代码结构已为后续接入后端做好准备。

## 技术栈

- **框架**：Vue 3（`<script setup>`）
- **构建工具**：Vite
- **UI 组件库**：Element Plus
- **路由**：Vue Router 4
- **HTTP 客户端**：Axios（已封装好，待接入真实后端）

## 目录结构（前端部分）

- `src/main.js`：应用入口，注册 Element Plus 和路由
- `src/App.vue`：根组件，仅渲染路由视图
- `src/router/index.js`：路由配置与简单前端路由守卫（基于本地 token + 角色）
- `src/utils/auth.js`：本地登录态工具（`token` 与 `userInfo` 的 `localStorage` 封装）
- `src/mock/achievement.js`：前端本地 mock 的成果数据与操作方法
- `src/services/achievementService.js`：**成果业务 service 层**，目前内部调用 mock，未来可平滑切换为真实接口
- `src/api/request.js`：Axios 实例封装（`/api` 前缀、统一响应处理）
- `src/api/auth.js` / `src/api/student.js` / `src/api/teacher.js`：按模块预先定义好的后端接口调用（暂未真正启用）
- `src/views/Login.vue`：登录页（前端模拟账号 + 角色选择）
- `src/views/student/*`：学生端页面（布局、成果列表、申报、编辑）
- `src/views/teacher/*`：教师端页面（布局、待审核列表、已审核列表）
- `src/style.css`：全局样式（管理后台风格、统一卡片与工具栏样式）

## 功能概览（当前前端实现）

- **登录模块（前端模拟）**
  - 仅校验账号 / 密码 / 角色是否为空，不校验真实性
  - 学生登录后跳转到「学生成果列表」，教师登录后跳转到「教师待审核列表」
  - 登录信息保存在 `localStorage` 中，刷新页面后仍保持登录态

- **学生端**
  - 成果申报：填写成果名称、类型、描述并提交（本地写入）
  - 成果列表：查看自己的成果，支持按类型 / 审核状态筛选、分页
  - 成果编辑 / 删除：仅对「待审核」成果开放编辑与删除
  - 成果详情：抽屉形式查看完整详情与审核结果

- **教师端**
  - 待审核列表：查看待审核成果，支持按类型筛选、分页
  - 审核操作：对成果执行「通过 / 驳回」，驳回需填写审核意见（≥5 字）
  - 已审核列表：查看已审核成果，支持按类型 / 审核结果筛选，展示审核意见与时间

> 当前版本只有学生、教师两个角色，管理员端及文件上传等功能在文档中已设计，代码层面预留了接口，尚未实现页面。

## 启动与开发

在 `frontend` 目录下：

```bash
npm install
npm run dev
```

默认启动地址一般为 `http://localhost:5173/`（以实际 Vite 输出为准）。

### 示例账号说明（前端模拟）

- 学生端：账号 `2023001`，密码 `123456`
- 教师端：账号 `teacher01`，密码 `123456`

登录仅做非空校验，以上账号为文档中的示例，方便演示。

## 与后端对接的准备情况

当前项目已经为将来接入真实后端做了以下准备：

- **接口文档**：`前端接口设计文档.md`
  - 定义了 `/api` 下的公共接口、学生模块、教师模块、管理员模块，以及返回结构规范和错误码约定。
- **Axios 封装**：`src/api/request.js`
  - 已按文档中的 `{ code, msg, data }` 响应结构进行统一处理。
- **业务 service 层**：`src/services/achievementService.js`
  - 页面**只依赖 service 层**，目前 service 内部调用 `mock` 数据。
  - 接入后端时，只需将 service 内的实现替换为 `src/api/student.js`、`src/api/teacher.js` 中的 `apiXXX` 方法即可，无需修改视图组件。
